使用教程
什么是批量文件处理？

批量文件处理是一种高效处理大量数据的方式。当你有大量的明文需要加密，或者有大量的密文需要查询解密时，逐个手动操作会非常繁琐。

通过上传一个 .txt 纯文本文件，系统可以自动：

  • 批量加密：将文件中的每一行明文转换为对应的 MD5 或 SHA1 哈希值，并存入数据库
  • 批量解密：自动识别每一行密文的格式（MD5/SHA1），查询数据库获取对应的明文

本功能特别适合：

  • 一次性处理大量用户密码的哈希存储
  • 批量验证已有的密文记录
  • 数据库迁移时的哈希数据整理
批量加密使用教程

第一步：准备文件

新建一个 .txt 纯文本文件，每行输入一条需要加密的明文。例如：

我的密码123
hello world
admin@2024
密码加密测试

注意：

  • 文件编码必须是 UTF-8
  • 每行只能有一条明文
  • 不建议使用过长的内容（建议不超过 100 字符）
  • 不要有空行

第二步：上传文件

点击"选择文件"按钮，或直接拖拽文件到上传区域。系统会显示文件名、大小和总行数。

第三步：选择加密算法

根据需求选择：

  • MD5：生成 32 位哈希值，速度快，兼容性广
  • SHA1：生成 40 位哈希值，安全性略高

第四步：开始处理

点击"开始处理"按钮，系统会逐行处理并实时显示进度。处理完成后会显示统计结果。

第五步：查看和下载结果

处理结果会显示在表格中，每条记录都可以单独复制。点击"下载结果"可以导出为文本文件。

批量解密使用教程

第一步：准备文件

将需要查询解密的密文放入 .txt 文件中，每行一条密文。例如：

e10adc3949ba59abbe56e057f20f883e
5baa61e4c9b93f3f0682250b6cf8331b7ee68fd8
da39a3ee5e6b4b0d3255bfef95601890afd80709

注意：

  • 每行必须是一个有效的 MD5（32位）或 SHA1（40位）哈希值
  • 密文可以是大小写混合，系统会自动处理
  • 格式错误的行会被标记为"无法识别"

第二步：上传文件

点击"选择文件"按钮上传密文文件。

第三步：选择解密模式

选择"批量解密"选项卡。系统会显示自动识别的说明文字。

第四步：开始处理

点击"开始处理"，系统会：

  1. 读取每一行密文
  2. 自动判断是 MD5（32位）还是 SHA1（40位）
  3. 查询数据库获取对应的明文
  4. 显示查询结果

结果说明

  • 解密成功：数据库中有该密文的记录，显示对应的明文
  • 解密失败：数据库中没有该密文的记录（从未加密存储过）
  • 格式错误：该行不是有效的 MD5 或 SHA1 格式
文件格式要求

支持的格式

文件类型 .txt（纯文本文件）
文件编码 UTF-8（推荐）、GBK、GB2312
文件大小 最大 10MB
每行内容 一条明文或一条密文
空行处理 会自动跳过空行

不支持的格式

  • Word 文档（.doc/.docx）
  • Excel 表格（.xls/.xlsx）
  • PDF 文件
  • 图片文件
  • 其他非文本格式

如何创建正确的 TXT 文件

方法一：使用记事本

  1. 打开 Windows 记事本
  2. 输入内容，每行一条
  3. 点击"文件" → "另存为"
  4. 编码选择"UTF-8"，保存为 .txt 文件

方法二：使用 VS Code

  1. 新建文件
  2. 输入内容
  3. 点击右下角编码，选择"通过编码保存"
  4. 选择 UTF-8，保存为 .txt 文件
加密/解密原理详解

哈希算法简介

哈希算法（Hash Function）是一种将任意长度的输入转换为固定长度输出的算法。它的特点包括：

  • 单向性：可以从明文计算出密文，但无法从密文反推明文
  • 固定输出：相同输入永远产生相同输出
  • 雪崩效应：输入微小变化会导致输出巨大变化

MD5 算法原理

MD5（Message-Digest Algorithm 5）是广泛使用的哈希算法：

  • 输出长度：128 位（32 个十六进制字符）
  • 设计者：RSA Data Security（1991 年）
  • 特点：计算速度快，兼容性好
  • 安全性：已被发现存在碰撞漏洞，不建议用于高安全场景
明文: "admin" MD5: "f0e4c2f76c58916ec258f246851bea0d" (32字符)

SHA1 算法原理

SHA1（Secure Hash Algorithm 1）是由 NSA 设计的安全哈希算法：

  • 输出长度：160 位（40 个十六进制字符）
  • 设计者：美国国家安全局（NSA）
  • 特点：比 MD5 更长，理论强度更高
  • 安全性：已被发现存在理论碰撞，同样不推荐高安全场景
明文: "admin" SHA1: "d033e22ae348aeb5660fc2140aec35850c4da997" (40字符)

自动识别原理

系统如何判断一串字符是 MD5 还是 SHA1？

判断逻辑: 1. 去除首尾空白 2. 转为小写 3. 判断长度: - 长度为 32 且为十六进制 → MD5 - 长度为 40 且为十六进制 → SHA1 - 其他 → 无法识别

数据库存储机制

当你在网页端进行加密时，系统会：

  1. 计算明文的哈希值
  2. 检查数据库是否已有该记录
  3. 如不存在，插入新记录（明文、密文、算法类型、时间戳）

批量处理时，系统会逐条处理，每条都会检查并存储到数据库。

为什么解密需要数据库？

由于哈希算法的单向性，无法从密文直接计算出明文。解密功能实际上是：

用户输入密文 → 数据库查询 → 返回对应的明文 例如： 数据库记录: { 明文: "admin", 密文: "f0e4c2f7...", 类型: "md5" } 用户查询: "f0e4c2f7..." 系统回答: "admin"

重要提醒：只有在数据库中已有记录的密文才能被"解密"，新建的密文需要先通过加密功能存储到数据库。

常见问题与注意事项

解密失败的原因

  • 数据库中没有该记录：密文从未通过加密功能存储到数据库
  • 算法类型不匹配：同一个明文用 MD5 和 SHA1 加密结果不同
  • 格式错误：输入的不是有效的 MD5（32位）或 SHA1（40位）格式

加密后如何解密？

加密完成后，系统会自动将每条记录存入数据库。此时你可以：

  1. 在结果表格中查看加密结果
  2. 点击"下载结果"保存到本地
  3. 复制单条结果进行分享

批量处理的限制

  • 单次处理最大 10MB：文件过大会导致浏览器卡顿
  • 仅支持 .txt 文件：其他格式需要先转换为纯文本
  • 仅支持电脑端：手机浏览器无法使用文件上传功能

数据安全说明

  • 所有数据存储在云端数据库中
  • 数据库仅允许插入和查询，不允许删除和修改
  • 请勿上传包含敏感信息的文件
  • 建议定期导出重要数据
