MD5 算法详解 MD5

MD5（Message-Digest Algorithm 5） 是 Ronald Rivest 于 1991 年设计的散列函数， 是上一代广泛使用的密码散列函数之一。MD5 将任意长度的输入消息处理生成 128 位（16 字节）的哈希值， 通常用 32 个十六进制字符表示。

输出长度
128 位 (16字节)
十六进制表示
32 个字符
设计者
Ronald Rivest
发布时间
1991 年
算法结构
Merkle-Damgård
分组长度
512 位 (64字节)

MD5 算法通过对输入消息进行填充、分块、迭代压缩，最终产生固定长度的摘要。 填充规则：先添加 bit"1"，然后补 0 直到消息长度 ≡ 448 (mod 512)，最后添加 64 位长度表示。

典型应用场景
数据库密码存储 文件完整性校验 数字签名 云存储校验 下载文件校验
安全警告： MD5 已被发现存在碰撞攻击和原像攻击的可行性。 不建议用于新的安全认证场景，如数字证书、SSL证书等。推荐使用更安全的 SHA-256 算法。
使用建议： MD5 仍适用于非安全敏感场景的文件完整性校验。 本工具会将每次加密的明文和密文存储到数据库，方便后续通过密文反查明文。

SHA1 算法详解 SHA1

SHA1（Secure Hash Algorithm 1） 是美国国家安全局 (NSA) 设计的安全散列标准， 是 SHA 家族中的第一个算法。SHA1 产生 160 位（20 字节）的哈希值，通常用 40 个十六进制字符表示， 比 MD5 多出 32 位，理论上提供更高的抗碰撞能力。

输出长度
160 位 (20字节)
十六进制表示
40 个字符
设计者
NSA (美国国安局)
发布时间
1995 年
算法结构
Merkle-Damgård
分组长度
512 位 (64字节)

SHA1 算法使用类似的 Merkle-Damgård 结构，但比 MD5 多了 5 个链接变量 (A-E)， 每轮使用 4 轮非线性函数和 20 次循环。输出更长带来更大的搜索空间， 理论上需要 2^80 次操作才能找到碰撞，比 MD5 的 2^64 次更安全。

典型应用场景
Git 版本控制系统 TLS/SSL 证书 HMAC 认证 PGP 邮件签名 代码签名
安全警告： SHA1 的碰撞攻击在 2017 年被实际验证可行（SHAttered 攻击）。 Google 和各大浏览器厂商已逐步废弃 SHA1 证书。重要安全场景请使用 SHA-256 或更高级别算法。
使用建议： SHA1 比 MD5 输出更长、更安全，但仍不建议用于高安全需求场景。 本工具会将每次加密的明文和密文存储到数据库，方便后续通过密文反查明文。

使用说明
  1. 在上方标签页选择 MD5 或 SHA1 算法
  2. 在输入框中输入需要加密的明文文本（支持中英文、数字、特殊字符）
  3. 点击 "加密并存储" 按钮生成哈希值
  4. 加密结果将显示在下方，同时自动存储到云端数据库
  5. 点击 "复制" 按钮可复制哈希值
  6. 后续可通过 "解密查询" 功能，输入密文反查明文

注意：所有加密操作在本地浏览器完成，密文自动上传到云端数据库存储。 相同明文重复加密会跳过重复存储，节省数据库空间。
