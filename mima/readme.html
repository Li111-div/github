工具简介

密码加密解密工具箱是一款功能完善的在线加密解密工具，提供多种哈希加密、可逆加密和编码转换功能。

哈希加密

MD5、SHA1 不可逆加密，输出固定长度哈希值

可逆加密

AES、DES、RC4 等12种对称加密算法

解密查询

通过密文查询数据库返回原始明文

离线工具

Base64、URL、进制等6种编码转换

MD5 加密功能

功能说明

MD5（Message-Digest Algorithm 5）是一种广泛使用的哈希算法，可以将任意长度的数据转换为 32 位（128 位）的哈希值。 MD5 是不可逆算法，相同输入始终产生相同输出，但无法从输出反推输入。

算法特点

  • 输出长度：32 位十六进制字符串（128 位）
  • 不可逆性：无法从密文反解明文
  • 雪崩效应：输入微小变化会导致输出完全不同
  • 碰撞抵抗：已发现可构造碰撞（不推荐用于安全场景）

应用场景

  • 数据完整性校验
  • 密码存储（建议加盐）
  • 文件校验
  • 数字签名

使用示例

输入: hello world
输出: 5eb63bbbe01eeed093cb22bb8f5acdc3

数据库存储

每次加密会自动将（明文、密文、算法类型）存储到云端数据库。 如果数据库中已存在相同记录，则不会重复存储，避免数据冗余。

SHA1 加密功能

功能说明

SHA1（Secure Hash Algorithm 1）是一种安全哈希算法，产生 40 位（160 位）的哈希值。 与 MD5 相比，SHA1 输出更长，理论上更安全，但同样不可逆。

算法特点

  • 输出长度：40 位十六进制字符串（160 位）
  • 不可逆性：无法从密文反解明文
  • 单向函数：设计为单向计算
  • 安全强度：已被发现存在理论攻击（SHA-2/SHA-3 更好）

应用场景

  • SSL/TLS 证书签名
  • Git 提交记录标识
  • 数字签名算法（如 DSA）
  • 数据完整性验证

使用示例

输入: hello world
输出: 2aae6c35c94fcfb415dbe95f408b9ce91ee846ed

解密查询功能

功能说明

解密查询功能通过输入密文和选择算法类型，在数据库中查找匹配的记录，返回原始明文。 支持 MD5 和 SHA1 两种算法的查询。

使用步骤

  1. 选择算法类型（MD5 或 SHA1）
  2. 输入要查询的密文（支持大小写自动转换）
  3. 点击"查询"按钮
  4. 查看返回的明文结果

注意事项

  • 必须先通过加密功能存储记录，才能查询到结果
  • MD5 和 SHA1 算法不互通，必须选择正确的算法
  • 输入时会自动过滤非十六进制字符
  • 大小写不敏感，会自动转换为小写查询

查询条件

数据库查询同时匹配（密文 + 算法类型），确保同一密文在不同算法下不会混淆。

可逆加密功能

可逆加密与哈希加密不同，使用密钥可以对数据进行加密和解密操作。 相同的密钥既可以加密也可以解密，这是可逆加密与不可逆哈希的本质区别。

支持的算法

推荐算法

AES-256 (CBC 模式)

高级加密标准，目前最安全的对称加密算法之一。支持 128、192、256 位密钥。本工具使用 256 位密钥。

  • 密钥长度：256 位（32 字节）
  • 分组长度：128 位（16 字节）
  • 安全性：极高，商业加密首选
  • 应用：TLS/SSL、文件加密、数据库加密
AES-ECB

电子密码本模式，是 AES 的最简单形式。相同的明文块会产生相同的密文块。

  • 特点：简单快速，但模式不安全
  • 注意：不推荐用于加密多块数据
AES-CTR

计数器模式，通过递增计数器生成密钥流，可以并行加密。

  • 特点：可并行处理，效率高
  • 应用：现代加密标准

经典算法

DES

数据加密标准，1977 年发布，是历史上最著名的对称加密算法。

  • 密钥长度：56 位（实际 64 位，含 8 位校验）
  • 分组长度：64 位
  • 安全性：已不安全，3DES 替代
DES-ECB

DES 的电子密码本模式，简单但存在与 ECB 相同的安全问题。

TripleDES

三重 DES，使用两个或三个不同的 DES 密钥对数据进行三次加密。

  • 密钥长度：168 位（3×56）或 112 位（2×56）
  • 安全性：比 DES 安全，但慢
  • 应用：旧系统兼容
TripleDES-ECB

TripleDES 的电子密码本模式。

流加密算法

RC4

序列密码，流加密的经典算法，曾广泛用于 SSL/TLS 和 WEP。

  • 密钥长度：可变，40-2048 位
  • 特点：简单快速
  • 注意：已发现弱点，不推荐新系统使用
RC4Drop

RC4 的改进版本，在生成密钥流前丢弃前 N 个字节，减少弱密钥的影响。

  • 丢弃字节：前 768-1536 字节
  • 安全性：比原始 RC4 更好
Rabbit

高性能流加密算法，2003 年提出，效率高，安全性好。

  • 密钥长度：128 位
  • 分组长度：64 位
  • 特点：速度快，硬件实现简单
Rabbit Legacy

Rabbit 的原始版本，保持向后兼容性。

其他算法

Blowfish

河豚算法，由 Bruce Schneier 设计，以安全性高和免费使用著称。

  • 密钥长度：可变，32-448 位
  • 分组长度：64 位
  • 特点：无专利，免费使用
  • 应用：加密工具、数据库

使用说明

  1. 输入明文：在输入框输入要加密的文本
  2. 输入密钥：输入加密密钥（DES 等算法有长度要求）
  3. 选择算法：从下拉菜单选择加密算法
  4. 点击加密：获取密文
  5. 点击解密：输入密文和相同密钥获取明文

重要提示

密钥管理：加密和解密必须使用相同的密钥，丢失密钥将无法解密！
算法选择：新项目推荐使用 AES-256，经典场景可选择 DES/TripleDES，流加密推荐 Rabbit。

离线编码工具

所有离线工具都在浏览器本地执行，不会向服务器发送任何数据。 适合处理敏感信息、开发调试等场景。

Base64 编码/解码

Base64 是一种基于 64 个可打印字符的编码方式，用于将二进制数据转换为 ASCII 文本。

  • 字符集：A-Z, a-z, 0-9, +, /（共 64 个字符）
  • 应用：电子邮件、URL 参数、数据传输
  • 示例：hello → aGVsbG8=

URL 编码/解码

URL 编码（百分号编码）用于将特殊字符转换为 URL 安全格式。

  • 空格编码：%20 或 +
  • 特殊字符：? → %3F，& → %26
  • 示例：hello world → hello%20world

进制转换

支持二进制、八进制、十进制、十六进制之间的相互转换。

  • 二进制 (BIN)：0-1，计算机底层表示
  • 八进制 (OCT)：0-7，常用于 Unix 权限
  • 十进制 (DEC)：0-9，人类日常使用
  • 十六进制 (HEX)：0-9, A-F，内存地址常用

支持自动识别：输入 0xFF 自动识别为十六进制，输入 0b11111111 自动识别为二进制。

Unicode 编码/解码

Unicode 是国际标准字符集，中文字符在 JavaScript 中以 \uXXXX 形式表示。

  • 中文字符：中 → \u4E2D
  • 表情符号：😀 → \u{1F600}
  • 应用：JSON 转义、JavaScript 字符串

HTML 实体编码/解码

将 HTML 特殊字符转换为实体形式，防止 XSS 攻击和渲染错误。

  • < → &lt;
  • > → &gt;
  • & → &amp;
  • " → &quot;
  • ' → &apos;

大小写转换

提供 5 种大小写转换方式，适应不同的文本处理需求。

  • 大写：hello → HELLO
  • 小写：HELLO → hello
  • 首字母大写：hello world → Hello World
  • 句首大写：hello. world. → Hello. World.
  • 大小写反转：Hello → hELLO

技术原理

哈希算法原理

哈希算法将任意长度的输入通过数学变换转换为固定长度的输出。 好的哈希算法具有以下特性：

  • 确定性：相同输入必产生相同输出
  • 单向性：无法从输出反推输入
  • 雪崩效应：输入微小变化导致输出巨大差异
  • 抗碰撞：难以找到两个不同输入产生相同输出

对称加密原理

对称加密使用相同的密钥进行加密和解密。加密过程和解密过程互为逆运算。 

加密: 明文 + 密钥 → 密文
解密: 密文 + 密钥 → 明文

技术栈

  • 前端：HTML5 + CSS3 + 原生 JavaScript
  • 样式：Tailwind CSS 暗黑风格
  • 加密库：CryptoJS（Web Crypto API 封装）
  • 数据库：Supabase（PostgreSQL 云数据库）
  • 图标：Font Awesome 6

注意事项

安全警告：MD5 和 SHA1 由于存在已知安全弱点，不推荐用于高安全场景的密码存储。 建议使用 bcrypt、Argon2 等专门的密码哈希算法。

使用规范

  • 本工具仅供学习和开发测试使用
  • 不要将重要密码或敏感信息存储在数据库中
  • 可逆加密务必妥善保管密钥，丢失将无法解密
  • Base64 和 Unicode 不是加密，只是编码格式
